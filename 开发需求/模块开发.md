写一个自己的工作用的 Agent，它将作为企业的公共大脑，记录同事们日常的工作流。
帮助大家把一些公共文件组织起来，梳理起来，一些业务流标准化，随时能通过这个公共大脑去查，新人来了以后降低磨合的周期。那它至少需要一些关系型数据库，还需要一些图数据库做业务流存储。需要一些硬盘服务器存储图纸、照片等实体数据资料。需要人员权限管控系统。任务发放与任务验收确认业务系统。
有时会有临时的成员加入进来，处理一些具体事务，这个Agent需要根据公司定好的业务流，逐步指导新成员逐步完成任务，并指导其提交标准成果与过程留痕
它没有前端WEB页面，只通过IM软件与员工沟通。
复杂的协同工作，能根据业务需求主动询问真人同事。或组织简单的html页面（问答选择与文档提交）与员工交互。
那这样的一个 Agent 它大概要用什么样的架构构成呢？现在有没有一些比较成熟的 Agent 框架呢？适合用来做实现这种需求的。
可使用astrbot作为基座。本项目做为一组独立Agent模块嵌入，主要打算利用其IM多频道桥接与harness基建。
尽量用成熟模块搭建，少重复造轮子。
